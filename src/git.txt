***Подготовительные операции***

Эти действия выполняются один раз на компьютере.

1. Скачать и установить Git - https://git-scm.com/install/
2. Удостовериться, что Idea "видит" установленный Git (File - Settings - Version Control - Git)
3. Если Idea не видит Git - указать вручную путь к программе при помощи кнопки в виде папки. Git по умолчанию устанавливается по этому пути - C:\Program Files\Git\bin\git.exe
4. Зарегистрироваться на GitHub (или использовать уже существующий аккаунт) - https://github.com/
5. Привязать Idea к аккаунту на GitHub (File - Settings - Version Control - GitHub)
6. Настроить терминал (File - Settings - Tools - Terminal), например, выбрать - C:\Program Files\Git\bin\bash.exe. На Mac OS можно обойтись без этого шага.
7. Зарегистрировать в Git ваши емейл и имя пользователя

````
git config --global user.email "your_email@example.com"
git config --global user.name "Your Name"
````

Последующие действия выполняются уже для каждого проекта в отдельности

***Публикация проекта на GitHub***

1. Создать проект
2. Публикация проекта (VCS - Share Project on GitHub)
3. Если операция не удаётся и выдаёт ошибку публичного ключа - Permission denied (publickey). В таком случае нужно выполнить в терминале команду - git remote set-url origin https://github.com/USER/REPO.git - эту ссылку нужно взять со страницы самого репозитория. Выполнить push.

***Скачать с GitHub проект на компьютер***

1. На сайте GitHub на вкладке Code проекта нажать зелёную кнопку Code и скопировать ссылку на проект.
2. В стартовом окне Idea нажать кнопку Clone Repository.
3. В строку URL вставить ссылку на проект, которую скопировали с сайта.
4. Нажать кнопку Clone.
5. Если клонируемый проект ещё не содержал ни одного класса, то после клонирования вручную создать папки src/main/java, и папку java пометить как папку исходного кода. ПКМ на папке java - Mark Directory as - Sources Root.

***Доработка проекта и обновление его на GitHub***

1. Пишем новый функционал.
2. Делаем операцию commit - фиксация текущего состояния проекта на локальной машине (при этом на GitHub пока ничего не выгружается - там хранится старая версия проекта)
3. Делаем операцию push - выгрузка текущих изменений в проекте на GitHub

***Доработка проекта в отдельных ветках***

Эти действия выполняются для каждой отдельно взятой задачи

1. Под конкретную задачу создаётся ветка от ветки master
2. Переключаемся на эту ветку (checkout), это можно сделать сразу при создании ветки
3. Пишем новый функционал
4. Делаем операцию commit, находясь в новой ветке
5. Делаем операцию push в новую ветку
6. Создаём Pull Request (запрос на слияние веток)
7. При необходимости вносим изменения в код и снова делаем commit push, при этом создавать новый PR не нужно - все изменения попадают в уже существующий PR
8. Выполняем слияние веток (проводим PR) - операция Merge
9. Удаляем более не нужную ветку с GitHub
10. На локальной машине переключаемся на ветку master
11. На локальной машине обновляем состояние проекта - Update Project
12. Удаляем ненужную ветку с локальной машины
